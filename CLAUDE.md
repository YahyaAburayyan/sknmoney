Copy `.env.example` to `.env.local` and fill in Supabase credentials:
```
NEXT_PUBLIC_SUPABASE_URL=...
NEXT_PUBLIC_SUPABASE_ANON_KEY=...
```

Apply SQL migrations in order via the Supabase SQL Editor:
1. `supabase/migrations/001_initial_schema.sql` — tables, trigger, indexes
2. `supabase/migrations/002_rls_policies.sql` — Row Level Security policies
3. `supabase/migrations/003_functions.sql` — `get_group_balances()` RPC function

## Architecture

**Stack:** Next.js 15 (App Router) · Supabase (Postgres + Auth + Realtime) · Tailwind CSS · Zod

### Route Groups
- `src/app/(auth)/` — unauthenticated pages (login, signup); centered card layout
- `src/app/(app)/` — authenticated app; layout fetches profile + group list for the sidebar
- `src/middleware.ts` — session refresh + redirect guard; protects `/dashboard` and `/groups/*`

### Data Flow Pattern
All mutations go through **Next.js Server Actions** (`src/actions/`). Pages are Server Components that query Supabase directly using `src/lib/supabase/server.ts`. The browser client (`src/lib/supabase/client.ts`) is only used in Client Components for Realtime subscriptions.

### Money Handling
All monetary values are **integer cents** throughout — in the database, in Server Actions, and in component state. Never store or compute as floats. Use `parseCurrencyInput()` and `formatCents()` from `src/lib/utils/currency.ts`. The `splitEqually()` helper assigns the rounding remainder to the first person (the payer).

### Multi-Tenancy / Security
Row Level Security is enforced at the Postgres layer via the `is_group_member(gid)` helper function. Users can only read/write data for groups they belong to. RLS must never be bypassed by using the service role key in client-facing code.

### Balance Algorithm
`src/lib/algorithms/debt-simplification.ts` contains two functions:
- `computeNetBalances()` — converts raw pair-wise balance rows (from the `get_group_balances` RPC) into per-person net amounts
- `simplifyDebts()` — greedy creditor-debtor matching; produces at most N-1 transactions for N people

### Key Conventions
- `src/types/database.ts` — hand-written DB types (replace with `supabase gen types typescript` once project is linked)
- `src/types/app.ts` — derived app-level types built on top of DB types
- Server Actions return `{ error: string } | { success: true }` — never throw; redirect after success
- Expenses use soft-delete (`deleted_at`) to preserve the audit trail; all balance queries filter `where deleted_at is null`
- Group invite codes are 6-character uppercase alphanumeric strings generated by Postgres
